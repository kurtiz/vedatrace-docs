Projects help you organize and isolate your logs. Each project has its own API keys, settings, and log stream.

## Creating a Project

1. Click on the **Projects** link in the sidebar
2. Click the **Create Project** button
3. Enter your project name
4. Click **Create**

## Switching Between Projects

If you have multiple projects, use the project switcher in the top navigation bar to switch between them:

All logs and settings are isolated per project.

## Project Settings

Each project can have:
- **Name** - Display name for your project
- **Description** - Optional description

## Deleting a Project

> ⚠️ **Warning:** Deleting a project is permanent and will delete all associated logs, API keys, and alert configurations.

To delete a project:
1. Go to the project settings
2. Click **Delete Project**
3. Confirm by typing the project name

## Best Practices

- Create separate projects for different environments (production, staging, development)
- Use descriptive project names to easily identify them
- Keep production and development logs separate for better organization
